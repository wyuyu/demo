中阶成员自愿参与，核心标准：你愿不愿意接手他写的代码？
务必做到实事求是，不用客气：别不忍心打低分，偏袒不利于他人。

基本流程

项目验收

  1. 百分百 UI 还原
  1. 百分百 功能实现

现场考核

  1. 找出项目 Bug 询问其解决办法
  1. 找一处功能，询问其实现思路以及可能出现的 Bug

评分细则

HTML CSS （10分）

百分百还原
  1. 一处 HTML Bug 扣两分
  1. 一处 CSS Bug 扣一分
  1. 中文命名或字母命名 一处扣一分
  1. 代码晦涩难读（未格式化） 扣三分
  1. 关键部分缺少注释（不得低于 10 处） 一处扣一分

JS （10分）

支持数据单目，加、减、乘、除 运算并正确显示
  1. HTML CSS 一处 Bug 扣一分
  1. 功能点都未实现 一处扣 一 分
  1. 中文命名或字母命名 一处扣一分
  1. 代码晦涩难读（未格式化） 扣三分
  1. 关键部分缺少注释（不得低于 10处）一处扣一分

综合评分低于 7 分，采取留阶

7 分以下的同学，有一次加试机会（自愿），满足要求即可进入中阶
